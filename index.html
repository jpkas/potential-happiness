Web Development PortFolio

By Jean-Pierre Kasonga

What html,css and Javascript can do ? I designed this simple page just to respond to the question as my web Development PortFolio. This web page is an example of how we can use html , css and javascript to make a responsive web page at some extent; knowing that bootstrap would have helped to meet the requirement of Mobile-First features. Only in coding one becomes a developer. That is how I discovered the power of div(s) to help organize the web page structure. The flow of my html page is based on a judicious choice of div(s). A first div contains the h1 and h2 elements indicating the title and the author. A secomd div contains the introducting words paragraph . Following is section element with 4 div(s) containing a list of things I have done ,two images of my portraits and a list of circumstances that called my attention to coding. After the central section I have a div for a table of things I like. Another div contains the concluding words. Finally I create a div to contain the fieldset with the form to be field by the client. Right this form doesn't work because this page is not linked to server. I noticed also that without CSS the html structure is vertical, each element mentioned above following the other without colors nor motion. For this reason ,I like to compare html page to the human skeleton without flesh. In art,the skeleton evokes the idea of ghost or dead person. I mean that html alone is boring. However, a human body without skeleton cannot stand. So html is needed and the developer has to set it up and embelish it. That is why, I have created a styles.css file that contains all the instructions to apply to my html page in order to make it look beautiful. To implement these instructions, I have chosen class(es) and Id(s) selectors to apply properties on the html elements. For example ,the class center allowed me to change the html vertical layout of lists and images into an horintal layout by the mean of the display-flex property. In the same way I applied color and background-color to diffrent parts of the page. Now the page looks like pretty. Yes. It is pretty but static,lacking motion or interactivity. Therefore , I invited Javascript to help. Instead of creating a Javascript file, I embeded my script in the html placing it before the body's closing tag. The Javascript code inside the script has the power to bring motion to the html-css-page. Of course ,I linked the 4 buttons to the id(s) of each element to be toggled. With the power of javascript addEventListener code the computer is not only able to listen to the click made upon the buttons, but to obey to that prompt. In obedience to the prompt , the computer will make the specified element appear or disappear from the page. So ,by clicking on the buttons one after another you will remove from the page the lists , images and table ; only to keep on it the buttons, the introducting and concluding texts and the form. Vis-versa, you can bring back these elements again just by clicking the same buttons one after another. It is almost magic,isn't?
